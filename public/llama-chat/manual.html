操作マニュアル

このページは、現在の Android 実装に基づく Dual AI Chat の操作ガイドです。設定項目や通信挙動は、実際のアプリ挙動に合わせて記載しています。

関連文書: プライバシーポリシー

1. 概要

Dual AI Chat は、ユーザーが指定した Ollama 互換サーバーと接続し、テキストチャット・音声入力・TTS・2 AI 掛け合い・Web Search を扱える Android アプリです。

  • Normal モード: 1 つのモデルとの通常チャット
  • Chatter モード: Base と Chatter Partner が交互に会話
  • 設定プロファイル、アバター、Debug Mode を端末内に保存

2. クイックスタート

  1. 起動時に表示される Quick Start は、「次回以降は表示しない」を選ぶまで表示されます。
  2. LLM tester with llama.cpp の API サーバー、または任意の Ollama 互換サーバーを用意します。
  3. 画面右上の ⚙ で設定を開き、Ollama URL に接続先を入力します。設定を閉じるときは同じボタンが 💾 に変わり、保存して閉じます。
  4. 起動時と設定画面を開いたときに /api/tags が呼ばれ、モデル一覧と接続状態タイルが更新されます。
  5. 必要なモデル、System Prompt、音声設定、Web Search 設定を入力したら保存してチャットに戻ります。

3. メイン画面の使い方

  • 入力欄 + Send: メッセージを送信します。
  • 入力欄が空のまま Send: Voice Input が有効なら音声認識を開始します。
  • 処理中の Send: ボタン表示が STOP に変わり、タップすると現在のリクエストを中止できます。
  • チャットパネル: 会話ログとアバター表示を確認できます。
  • ログエリア上部のハンドル: チャットエリアの拡大/縮小を切り替えます。

4. 設定の見どころ

セクション 内容
General 表示言語、Ollama URL、接続状態タイル、Streaming、TTS、Voice Input、Auto Voice Input、ユーザー名、Config Profile を設定します。
Chat Normal / Chatter モード、History Limit、おしゃべり間隔、Base / Chatter Partner のモデル・名前・音声・System Prompt を設定します。
Expert Web Search、検索 API URL / API Key、Web Search Model、Debug Mode を設定します。
Config Profile 設定プロファイルの保存、SELECT Template による読込、削除を行います。アバター画像もプロファイルと一緒に扱われます。

5. モードと会話履歴

  • Normal: Base 側のモデルで通常の会話を行います。
  • Chatter: Base と Chatter Partner が交互に応答します。間隔は Chatter Interval で設定します。
  • History Limit: リクエスト時に送る過去メッセージ数を制御します。
  • Reset Conversation Log: 現在セッション中の会話ログをリセットします。
  • 現在の実装では会話履歴はメモリ上で保持され、アプリ再起動後に復元される永続保存は行われません。

6. 音声機能

  • Voice Input: 音声認識を使って入力できます。アプリはまずオフライン認識を優先し、必要に応じてオンライン認識へフォールバックします。
  • Text-to-Speech: 応答を文単位で読み上げます。
  • Auto Voice Input: 応答完了後、条件が整えば次の音声入力を自動で開始します。
  • Speech Language / Rate / Pitch: Base と Chatter Partner それぞれで独立して設定できます。

7. Web Search と Debug Mode

  • Web Search を有効にし、API Key を設定すると、検索クエリを指定 API に送信して結果を会話コンテキストへ追加できます。
  • Brave Search API 形式の URL を設定した場合は Brave 向けの処理を使います。
  • Web Search Model は /api/tags で取得したモデル一覧から選択します。
  • Debug Mode を有効にすると、API リクエスト / レスポンスの詳細ログを画面上で確認できます。

8. アバターと設定プロファイル

  • アバターは c0（背景）/ c1（基本）/ c2（まばたき）/ c3（会話中）の 4 スロット構成です。
  • Base と Chatter Partner で別々の画像を設定できます。
  • Clear を押すと既定状態に戻せます。
  • 設定プロファイルを保存すると、設定値と対応するアバター画像バックアップをまとめて再利用できます。

9. 補足

  • 接続エラー時は、Ollama URL の状態タイルと /api/tags の取得結果を確認してください。
  • 音声認識が動かない場合は、マイク権限と端末の音声認識サービスの利用可否を確認してください。
  • 検索 API やリモートサーバーを使う場合、送信データはそれぞれの提供者側ポリシーに従って処理されます。
