User Manual

This page describes how the current Android implementation of Dual AI Chat works, based on the actual app behavior.

Related doc: Privacy Policy

1. Overview

Dual AI Chat is an Android app that connects to a user-configured Ollama-compatible server and combines text chat, voice input, TTS, two-AI chatter, and optional Web Search.

  • Normal mode: standard chat with one model
  • Chatter mode: Base and Chatter Partner alternate turns
  • Profiles, avatars, and Debug Mode are stored locally on the device

2. Quick Start

  1. The Quick Start popup is shown until you choose “Do not show again”.
  2. Prepare either the API server from LLM tester with llama.cpp or another Ollama-compatible server.
  3. Open settings with the ⚙ button, enter your server in Ollama URL, and close settings with the same button after it changes to 💾.
  4. On startup and whenever settings are opened, the app calls /api/tags to refresh model lists and the status tile.
  5. Set the model, system prompt, speech options, and optional Web Search values, then save and return to chat.

3. Main Screen Behavior

  • Input + Send: Sends the current message.
  • Empty input + Send: If Voice Input is enabled, speech recognition starts instead.
  • Send while processing: The button changes to STOP and can cancel the active request.
  • Chat panel: Shows message history and avatar state.
  • Log-area handle: Expands or collapses the chat panel.

4. Settings at a Glance

Section What it controls
General Display language, Ollama URL, connection-status tile, Streaming, TTS, Voice Input, Auto Voice Input, user name, and Config Profile actions.
Chat Normal / Chatter mode, History Limit, chatter interval, plus model/name/speech/system prompt settings for Base and Chatter Partner.
Expert Web Search, search API URL / API key, Web Search Model, and Debug Mode.
Config Profile Save profiles, load them through SELECT Template, and delete them. Avatar images are carried with the profile data.

5. Modes and Conversation Memory

  • Normal: Standard one-model chat using the Base settings.
  • Chatter: Base and Chatter Partner talk to each other. The interval is controlled by Chatter Interval.
  • History Limit: Controls how many recent messages are included when the next request is sent.
  • Reset Conversation Log: Clears the current session conversation memory.
  • In the current implementation, conversation history is kept in memory during the session and is not restored after an app restart.

6. Voice Features

  • Voice Input: Uses speech recognition for message input. The app prefers offline recognition first and can fall back to online recognition if needed.
  • Text-to-Speech: Reads responses aloud sentence by sentence.
  • Auto Voice Input: Can automatically start the next voice-input cycle after a response finishes.
  • Speech Language / Rate / Pitch: Can be configured independently for Base and Chatter Partner.

7. Web Search and Debug Mode

  • If Web Search is enabled and an API key is configured, the app sends the query to the configured search API and adds the results to the chat context.
  • If the configured URL is Brave-compatible, the app uses Brave-specific request formatting.
  • Web Search Model is chosen from the list returned by /api/tags.
  • Debug Mode shows detailed API request and response logs inside the app.

8. Avatars and Profiles

  • Each avatar uses four slots: c0 (background), c1 (base), c2 (blink), and c3 (talking).
  • You can set separate image sets for Base and Chatter Partner.
  • Clear resets a slot back to its default state.
  • When you save a settings profile, the related avatar backups are saved with it for later reuse.

9. Notes

  • If you cannot connect, check the Ollama URL status tile and whether /api/tags is reachable.
  • If voice recognition does not start, check microphone permission and whether speech recognition is available on the device.
  • If you use remote servers or third-party search APIs, those services process the relevant data under their own policies.
