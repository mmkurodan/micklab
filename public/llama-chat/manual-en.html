User Manual

This page documents how to use Dual AI Chat based on the current Android implementation.

Related doc: Privacy Policy

■ Quick Start

  • At startup, the app shows a Quick Start popup unless you choose Do not show again.
  • Before chatting, install and start LLM Tester with llama.cpp or Ollama, then enable the API server.
  • In Settings, use Launch LLM API above the API status tile. If LLM Tester is missing, the Play Store page opens.

■ Screen

  • Tap ⚙️ to open settings, then 💾 to save and close.
  • Tap the top handle of the log area to expand/collapse the chat panel.

■ Settings

  • Language: Choose the app language from the dropdown as English or 日本語.
  • Launch LLM API: Use Launch LLM API above the API status tile to launch LLM Tester.
  • API Status: The tile under Ollama URL shows the /api/tags status.
  • Connection Settings: Use Settings to configure connection settings and avatar images.
  • Templates: Use SELECT Template under Config Profile to load templates.
  • Updates: On startup and each time settings opens, /api/tags refreshes model lists and the API status tile.

■ Sending

  • Enter a message and press Send.
  • If Voice Input is enabled, pressing Send with empty input starts voice input.
  • While processing, the button shows STOP and can cancel the request.
  • Note: On the first message, approximately 1GB of data may be downloaded for model downloads by the LLM API, which can take time.

■ Reset

  • To reset the conversation, press Reset Conversation Log in Settings.

■ Modes

  • Normal: Chat with a single model.
  • Chatter: Base and Chatter Partner alternate turns.
  • Chatter Interval: Sets the delay between turns (seconds).

■ Settings Tabs

  • Use the Base/Chatter Partner tabs to switch settings.
  • Configure Model/Name/System Prompt/Speech Language/Rate/Pitch.

■ Response

  • Streaming: Shows responses in real time.
  • Text-to-Speech: Reads responses aloud.
  • History Limit: Controls how many past messages are sent.

■ Expert Settings

  • Web Search: Enable to use the configured search API.
  • Web Search Model: Selected from /api/tags list (default: default).
  • Brave URLs: Endpoints use Brave-optimized search handling.
  • Debug Mode: Shows API request/response logs.

■ Avatar

  • c0: Background
  • c1: Base
  • c2: Blink
  • c3: Talking
  • You can set different images for Base and Chatter Partner.
  • Press Clear to reset to default.

■ Other

  • Settings, templates, popup preferences, and images are stored locally on the device.
