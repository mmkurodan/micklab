LLM tester with llama.cpp

LLM tester with llama.cpp は、LLMの挙動を「測る・比較する・再現する」ためのローカルLLM向け実験ツールです。スマホ上でモデル比較、パラメータ検証、テンプレート検証が行えます。

文書: 操作マニュアル | プライバシーポリシー

  • 「LLMの“挙動”を可視化する。」 — 応答品質・速度・安定性を精密に測定する実験環境。
  • 「ローカルLLMを、もっと正確に理解する。」 — モデル選択から生成ログまで、すべてを一つの画面で管理できます。
  • 「モデル比較・テンプレ検証・パラメータ実験。すべて一つで。」 — 研究者・開発者向けの高度な機能を提供します。
スクリーンショット

主な特徴

  • 複数モデルの比較: 同一プロンプトで応答を並列比較し、モデルごとの違いを把握できます。
  • 詳細なログと再現性: 生成パラメータ、テンプレート、リクエスト/レスポンスを記録し、実験の再現性を確保します。
  • テンプレート検証: ChatML / Qwen / LLaMA 等のテンプレート差異を簡単に比較検証できます。
  • 高度なサンプリング設定: Temperature、Top‑p、Mirostat、DRY などの影響を試験可能です。
  • スマホ単体で完結: Android端末だけでモデルの挙動を分析できる軽量なテスト環境です。
  • 研究者向けの透明性: デバッグモードでAPIのやり取りを可視化し、挙動の原因分析を支援します。
API仕様（端末内 Ollama 互換サーバー）
  • GET /api/tags（POST /api/tags も可）: 設定済みモデル一覧を返します。レスポンス例: {"models":[{"name":"default","model":"default","modified_at":"2026-03-15T00:00:00Z","size":0,"details":{"format":"gguf","family":"llama","parameter_size":"unknown","quantization_level":"unknown"}}]}
  • POST /api/generate: 本文JSONは model（省略時 default）, prompt, system（任意）, stream（省略時 true）。stream=true は NDJSON チャンク（response を逐次返却、最終チャンクで done:true）、stream=false は単一JSON（response 全文 + done:true）です。
  • POST /api/chat: 本文JSONは model（省略時 default）, messages（必須: role/content配列）, stream（省略時 true）。stream=true は NDJSON チャンク（message.content を逐次返却、最終チャンクで done:true）、stream=false は単一JSON（assistant の全文応答 + done:true）です。
  • 共通仕様: 同時生成は1件のみ。ビジー時は待機キュー最大10件・最大60秒で待機し、超過時は 503 を返します。主なエラーは 400（不正JSON/メッセージ不足）, 500（設定読み込み失敗）, 404（未定義エンドポイント）です。
利用上の注意

生成の速度や安定性はモデルや端末性能に依存します。
