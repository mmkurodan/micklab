User Manual

This public manual summarizes how the current Android implementation of LLM tester with llama.cpp works.

Related doc: Privacy Policy

1. Overview

LLM tester with llama.cpp is a local LLM testing app for Android that combines prompt input, inference tuning, template inspection, logging, and an optional Ollama-compatible API in one place.

  • Built with Llama (llama.cpp)
  • Supports both URL-based model loading and local GGUF import
  • Can expose /api/chat, /api/generate, and /api/tags when the built-in API server is enabled

2. Before You Start

  • Large model downloads can require gigabytes of data. Wi‑Fi is recommended instead of cellular data.
  • On first launch, the app may show Quick Start and API enablement prompts. Both support a “Don’t show next time” option.
  • While inference is busy, the Settings button is disabled and automatically re-enabled when processing finishes.

3. Recommended Setup

  1. Open Settings from the main screen.
  2. In Model Selection, enter a model URL or import a local .gguf file.
  3. Press Load Model. For local import, the picker opens in Downloads by default.
  4. Adjust parameters if needed, then save the configuration with Save Config.
  5. Press SAVE & CLOSE to persist the settings and apply them immediately to the current model.

Reachable HTTP/HTTPS model URLs are supported. HTTPS uses standard SSL/TLS certificate verification.

4. Main Screen

  • Enter Prompt: Type the prompt you want to run.
  • Send: Starts generation. If the model is not already loaded, the app tries to load the model for the current configuration first.
  • Settings: Disabled while inference is busy. Detailed settings live on a separate screen.
  • Re-init Model: Stops the active job and reinitializes the current profile.
  • View Log / Hide Log: Shows the latest 100 log lines in the output area, then toggles back to normal output.
  • Update: Reloads the log view.
  • Clear Log: Clears the log file.
  • Start/Stop API Server: Toggles the built-in API server.
  • Copy / Save: Copies or saves the current response, or the displayed log content.
  • Processing Status / Logs: Shows timestamped processing logs.

5. Settings Screen

  • Configuration Management: Save, load, and delete configurations. A default default profile is created automatically.
  • Model Selection: Load from a URL or import local .gguf files. Imported files are copied into app storage and only their filenames are stored in settings.
  • Model Parameters: Configure basic parameters, penalties, Mirostat, extra sampling controls, DRY, and GPU Offload Layers.
  • Prompt Template: Set System Prompt, Think on/off, and a custom chat template. If no custom template is set, the app auto-selects one based on model family.
  • Llama API Server: Set the listening port. Start/stop is handled from the startup popup or the main screen.
  • Display Language: Switch between Japanese and English. The first launch follows the device locale and later launches reuse your saved choice.
  • Log Settings / Show License / Documents: Choose log level, view license text, and open the in-app documents screen.
  • SAVE & CLOSE / CLOSE: Save and apply immediately, or leave without saving.

6. Parameter Groups at a Glance

Group Main items Purpose
Basic n_ctx, n_threads, n_batch, GPU Offload Layers, temp, top_p, top_k Controls speed, memory usage, and the overall response style.
Penalty penalty_last_n, penalty_repeat, penalty_freq, penalty_present Helps suppress repetition and control how already-used tokens are treated.
Mirostat mirostat, mirostat_tau, mirostat_eta Enables automatic consistency control during sampling.
Additional sampling min_p, typical_p, dynatemp_range, dynatemp_exponent, xtc_probability, xtc_threshold, top_n_sigma Provides finer control when you want to study or tune response behavior in detail.
DRY dry_multiplier, dry_base, dry_allowed_length, dry_penalty_last_n, dry_sequence_breakers Useful for reducing repetitive patterns in longer generations.
Output / Think streaming, enableThinking, systemPrompt, customChatTemplate Controls real-time output, visible reasoning behavior, and template differences.

7. Prompt Templates

If the custom chat template is empty, the app estimates the model family from the filename and auto-selects a suitable template.

  • Supported families: Gemma, Qwen, Mistral, LLaMA, Phi, Zephyr, Hermes
  • Fallback when unknown: ChatML
  • Gemma-family formatting preserves the system / user / model order.
  • The selected template is logged in Processing Status / Logs and INFO-level logs.

8. Built-in API Server

  • Endpoints: /api/chat, /api/generate, /api/tags
  • Only one generation can run at a time. When busy, up to 10 requests are queued for up to 60 seconds.
  • Queue overflow or timeout returns 503.
  • Android 13 and later may require notification permission.

9. Notes

  • Very large models may fail to load because of address-space or memory limits, or because a load was interrupted. If that happens, try a smaller model or load it again from Settings.
  • Logs and crash information remain on the device and can be reviewed or removed locally.
