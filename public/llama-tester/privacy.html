日本語

1. 適用範囲

本ポリシーは、Mick Lab が提供する Android アプリ「LLM tester with llama.cpp」に適用されます。

2. 端末内に保存される情報

本アプリは、以下の情報をユーザーの端末内に保存します。

  • 設定情報（モデル URL、各種推論パラメータ、API ポート、ログレベル、表示言語など）
  • 構成ファイル（configs/*.json）
  • ダウンロードまたは取り込んだモデルファイル
  • ログファイルおよびクラッシュ関連ファイル（ollama.log、last_crash.txt、native_crash.txt）

3. ネットワーク通信

  • モデルを URL から取得する場合、アプリはユーザーが指定した URL に通信します。
  • ローカル .gguf ファイルを取り込む場合、選択ファイルはアプリ保存領域へコピーされ、外部サーバー通信は行いません。
  • アプリ内の通常のプロンプト生成は端末内で処理され、開発者へ送信されません。
  • 内蔵の Ollama 互換 API サーバーを有効にした場合、アプリは 0.0.0.0 で待機し、同一端末またはローカルネットワーク内からのリクエストを受け付けます。
  • この API に送られたプロンプトやメッセージ本文は、端末内またはローカルネットワーク内で処理され、開発者へ送信されません。
  • HTTPS URL を使用する場合、通常の SSL/TLS 証明書検証が行われます。ローカル API を HTTP で利用する場合、通信は暗号化されないことがあります。

4. 利用目的

上記データは、モデルの読み込み、推論の実行、設定の再利用、ログ表示、API 提供など、アプリ機能を実現するためにのみ使用されます。

5. ログと保持期間

ログにはアプリの動作状況や、機能利用に応じた API 関連情報が含まれる場合があります。これらは端末内にのみ保持され、ユーザーが削除するかアプリデータを消去するまで保存されます。ログはメイン画面の Clear Log から削除できます。

6. 開発者による取得について

開発者は、ユーザーのプロンプト、モデル出力、ログ、モデルファイルを受信しません。本アプリは広告 SDK や解析 SDK を利用してこれらの情報を開発者へ送信する実装にはなっていません。

7. お問い合わせ

ご不明点がある場合は、以下までお問い合わせください。
micklab2026@gmail.com

English

1. Scope

This policy applies to the Android application “LLM tester with llama.cpp” provided by Mick Lab.

2. Data Stored on Your Device

The app stores the following information locally on the user’s device:

  • Configuration data, such as model URLs, inference parameters, API port, log level, and display language
  • Configuration files (configs/*.json)
  • Downloaded or imported model files
  • Log and crash-related files (ollama.log, last_crash.txt, and native_crash.txt)

3. Network Communication

  • When you download a model from a URL, the app connects only to the URL you specify.
  • When you import a local .gguf file, the selected file is copied into app storage and no external server communication occurs.
  • Normal prompt generation inside the app is processed on-device and is not sent to the developer.
  • If you enable the built-in Ollama-compatible API server, the app listens on 0.0.0.0 and accepts requests from the same device or the local network.
  • Prompts and message bodies sent to that API are processed on-device or within the local network and are not transmitted to the developer.
  • For HTTPS model URLs, standard SSL/TLS certificate verification is used. If you use the local API over HTTP, that traffic may be unencrypted.

4. Purpose of Use

The data above is used only to provide app functionality such as model loading, inference, saved configurations, log display, and the built-in API.

5. Logs and Retention

Logs may contain operational information and, depending on the feature used, API-related details. They remain on the device only and are retained until the user deletes them or clears the app data. Logs can be removed with Clear Log on the main screen.

6. What the Developer Receives

The developer does not receive your prompts, model outputs, logs, or model files. The current app implementation does not send this information to the developer through advertising or analytics SDKs.
