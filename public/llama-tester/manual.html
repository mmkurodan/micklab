操作マニュアル

このページは、現在の Android アプリ実装に合わせて、LLM tester with llama.cpp の使い方と挙動を整理した公開マニュアルです。

関連文書: プライバシーポリシー

1. 概要

LLM tester with llama.cpp は、Android 端末上で GGUF モデルを実行し、プロンプト入力・推論パラメータ調整・テンプレート確認・ログ確認・Ollama 互換 API 提供までを一つのアプリで扱えるローカルLLM検証ツールです。

  • Built with Llama (llama.cpp)
  • モデルは URL からの取得とローカルファイル取り込みの両方に対応
  • 必要に応じて /api/chat / /api/generate / /api/tags を提供

2. 使い始める前に

  • 大きなモデルのダウンロードには数 GB 単位の通信が必要になる場合があります。モバイル回線ではなく Wi‑Fi の利用を推奨します。
  • 初回起動時には Quick Start と API サーバー有効化ポップアップが表示される場合があります。どちらも「次回以降は表示しない」を選べます。
  • 推論中（Busy）は Settings ボタンが無効化され、処理完了後に自動で再度有効になります。

3. 推奨セットアップ

  1. メイン画面で Settings を開きます。
  2. Model Selection で、モデル URL を入力するか、ローカル端末から .gguf ファイルを取り込みます。
  3. Load Model でモデルを読み込みます。ローカル取り込み時、ファイルピッカーは Downloads を既定で開きます。
  4. 必要に応じてパラメータを調整し、Save Config で構成を保存します。
  5. SAVE & CLOSE を押すと設定が保存され、その時点のモデルへ即時反映されます。

HTTP/HTTPS のモデル URL を使用できます。HTTPS では通常の SSL/TLS 証明書検証が行われます。

4. メイン画面

  • Enter Prompt: プロンプトを入力します。
  • Send: 生成を開始します。モデル未ロード時は、現在の構成に対応するモデルのロードを自動で試みます。
  • Settings: 推論中は無効化されます。設定パネルではなく別画面で詳細設定を行います。
  • Re-init Model: 進行中の生成を停止し、現在のプロファイルを再初期化します。
  • View Log / Hide Log: モデル出力欄にログファイルの最新 100 行を表示し、再度押すと通常の出力へ戻ります。
  • Update: ログ表示中の内容を再読み込みします。
  • Clear Log: ログファイルを空にします。
  • Start/Stop API Server: 内蔵 API サーバーを起動または停止します。
  • Copy / Save: モデル出力またはログ表示内容をコピー・保存します。
  • Processing Status / Logs: タイムスタンプ付きの処理ログを表示します。

5. 設定画面

  • Configuration Management: 設定の保存・読み込み・削除を行います。既定の default 構成が自動生成されます。
  • Model Selection: URL からの取得に加えて、ローカル端末の .gguf を取り込めます。取り込んだファイルはアプリ保存領域へコピーされ、設定欄にはファイル名のみが保存されます。
  • Model Parameters: 基本パラメータ、Penalty、Mirostat、追加サンプリング、DRY、GPU Offload Layers を設定できます。
  • Prompt Template: System Prompt、Think 有効/無効、custom chat template を設定できます。未指定時はモデルファミリーから自動選択されます。
  • Llama API Server: API の待受ポートを設定できます。起動/停止は起動時ポップアップまたはメイン画面から行います。
  • Display Language: 日本語 / English を切り替えられます。初回は端末言語をもとに決まり、選択内容は保存されます。
  • Log Settings / Show License / Documents: ログレベル、ライセンス表示、文書画面への移動を行います。
  • SAVE & CLOSE / CLOSE: 保存して即時適用するか、保存せず閉じるかを選べます。

6. パラメータ群の見どころ

項目群 主な項目 用途
基本 n_ctx, n_threads, n_batch, GPU Offload Layers, temp, top_p, top_k 速度、メモリ使用量、応答の多様性や安定性を調整します。
Penalty penalty_last_n, penalty_repeat, penalty_freq, penalty_present 同じ語句の繰り返しや、出現済みトークンの扱いを調整します。
Mirostat mirostat, mirostat_tau, mirostat_eta 出力の一貫性を自動調整するサンプリング手法を切り替えます。
追加サンプリング min_p, typical_p, dynatemp_range, dynatemp_exponent, xtc_probability, xtc_threshold, top_n_sigma モデルの出力傾向をさらに細かく追い込みたいときに使います。
DRY dry_multiplier, dry_base, dry_allowed_length, dry_penalty_last_n, dry_sequence_breakers 長文生成での単調な繰り返しを抑えたいときに有効です。
出力 / Think streaming, enableThinking, systemPrompt, customChatTemplate ストリーミング表示、思考出力の扱い、テンプレート差分の検証に使います。

7. プロンプトテンプレート

custom chat template が空の場合、アプリはモデルファイル名からファミリーを推定し、適切なテンプレートを自動選択します。

  • 対応ファミリー: Gemma, Qwen, Mistral, LLaMA, Phi, Zephyr, Hermes
  • 未判定時のフォールバック: ChatML
  • Gemma 系では system / user / model の並びを維持します。
  • 選択結果は Processing Status / Logs と INFO レベルログに記録されます。

8. 内蔵 API サーバー

  • 提供エンドポイント: /api/chat, /api/generate, /api/tags
  • 同時生成は 1 件のみです。ビジー時は最大 10 件までキューに入り、最大 60 秒待機します。
  • キュー満杯または待機超過時は 503 を返します。
  • Android 13 以上では通知権限が必要になる場合があります。

9. 補足

  • 大きなモデルの読み込みは、アドレス空間不足や中断操作で失敗する場合があります。その場合は小さいモデルを試すか、Settings から再度読み込んでください。
  • ログやクラッシュ情報は端末内に保持され、必要に応じて確認や削除ができます。
